### **物理层**
| 协议/标准/技术 | 核心功能与考点 |
|:---|:---|
| **EIA/TIA-232, EIA/TIA-449** | 物理接口标准（机械、电气、功能特性）。 |
| **V.24, V.35** | ITU-T制定的DTE/DCE接口标准。 |
| **RJ-45** | 双绞线以太网物理接口。 |
| **光纤接口（如SC, ST, LC）** | 用于光纤连接的物理接口。 |
| **PCM (脉码调制)** | 将模拟信号数字化的技术，是T1/E1等数字传输的基础。 |
| **SONET/SDH** | 光纤同步传输标准。 |
| **IEEE 802.3 (物理层部分)** | 以太网物理层标准（如10BASE-T, 100BASE-TX, 1000BASE-LX）。 |
| **IEEE 802.11 (物理层部分)** | 无线局域网物理层标准（如FHSS, DSSS, OFDM）。 |

**考查重点**：接口特性、编码方式（如曼彻斯特编码）、奈奎斯特与香农定理。

### **数据链路层**
| 协议/技术 | 核心功能与考点 |
|:---|:---|
| **PPP (点对点协议)** | 在点对点链路上封装数据报，支持身份验证（PAP, CHAP），是广域网接入常用协议。 |
| **HDLC (高级数据链路控制)** | 面向比特的同步链路层协议，PPP的前身。 |
| **CSMA/CD** | 载波监听多路访问/碰撞检测，**有线以太网**的核心介质访问控制协议。 |
| **CSMA/CA** | 载波监听多路访问/碰撞避免，**无线局域网 (IEEE 802.11)** 的核心MAC协议。 |
| **以太网 (IEEE 802.3)** | 局域网标准，定义了帧格式（MAC帧）、MAC地址、交换机工作原理。 |
| **VLAN (虚拟局域网) - IEEE 802.1Q** | 在交换机上划分逻辑广播域，需掌握其帧标记原理。 |
| **STP (生成树协议) - IEEE 802.1D** | 在交换网络中防止二层环路，计算无环路径。 |
| **ARP (地址解析协议)** | 将IP地址解析为MAC地址，**逻辑上属于网络层，但常在此层讨论**，是IP与MAC关联的关键。 |
| **交换机** | 核心设备，基于MAC地址进行转发，需理解其学习、转发、过滤过程。 |

**考查重点**：以太网帧格式、CSMA/CD原理（最小帧长、碰撞域）、PPP协商过程、交换机与VLAN。

### **网络层**
| 协议/技术                | 核心功能与考点                                                                                                                                                        |
| :------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **IP (网际协议)**        | 提供无连接、不可靠的数据报传输服务，重点掌握IPv4分组格式、分片机制。                                                                                                                           |
| **IPv6**             | 下一代IP协议，解决地址耗尽问题，重点掌握与IPv4的区别、地址表示法、过渡技术。                                                                                                                      |
| **ICMP (互联网控制报文协议)** | 报告差错和询问，如Ping（回显请求/应答）、Traceroute（超时报文）。                                                                                                                       |
| **ARP (地址解析协议)**     | 再次强调，它是IP层依赖的关键协议。                                                                                                                                             |
| **RARP (逆地址解析协议)**   | 已被DHCP取代，了解即可。                                                                                                                                                 |
| **NAT (网络地址转换)**     | 将私有IP转换为公网IP，解决IPv4地址短缺问题，需掌握NAPT。                                                                                                                             |
| **路由选择协议**           | **内部网关协议 (IGP)**：<br>- **RIP**：基于距离向量，以“跳数”为度量，周期性广播路由表。<br>- **OSPF**：基于链路状态，使用Dijkstra算法，收敛快，支持分区域。<br>**外部网关协议 (EGP)**：<br>- **BGP**：路径向量协议，用于自治系统间，保证路由无环。 |
| **SDN (软件定义网络)**     | 控制与数据平面分离，流表机制（OpenFlow），参考前次补充内容。                                                                                                                             |
| **路由器**              | 核心设备，基于IP地址进行转发，需理解其分组转发流程（最长前缀匹配）。                                                                                                                            |

**考查重点**：IP地址分类与子网划分、CIDR、路由算法（距离向量与链路状态对比）、NAT原理、SDN基本概念。

### **运输层**
| 协议 | 核心功能与考点 |
|:---|:---|
| **TCP (传输控制协议)** | **面向连接、可靠、字节流**服务。关键机制：三次握手/四次挥手、滑动窗口、流量控制、拥塞控制（慢开始、拥塞避免、快重传、快恢复）、校验和、序号与确认号。 |
| **UDP (用户数据报协议)** | **无连接、不可靠、数据报**服务。特点：首部开销小、速度快、支持一对多。常用于DNS、视频、语音、TFTP、RIP。 |

**考查重点**：TCP可靠传输机制、流量控制与拥塞控制的区别与算法、三次握手/四次挥手过程、TCP与UDP的对比。

### **应用层**
| 协议 | 核心功能与考点 |
|:---|:---|
| **HTTP (超文本传输协议)** | Web的核心，**无状态**，基于TCP。重点掌握HTTP/1.0（非持续连接）、HTTP/1.1（持续连接）、请求/响应报文格式、常见状态码（200， 301， 404）。 |
| **HTTPS (HTTP over SSL/TLS)** | HTTP的安全版本，提供加密、身份认证和完整性保护。 |
| **FTP (文件传输协议)** | 基于TCP，使用**两个连接**：控制连接（端口21）和数据连接（端口20或随机）。 |
| **SMTP (简单邮件传输协议)** | 用于**发送**邮件，基于TCP，端口25。使用“推”的通信方式。 |
| **POP3 (邮局协议第3版)** | 用于**接收**邮件，基于TCP，端口110。使用“拉”的通信方式，将邮件下载到客户端。 |
| **IMAP (互联网消息访问协议)** | 功能比POP3更强大的邮件接收协议，允许在服务器上管理邮件，端口143。 |
| **DNS (域名系统)** | **最重要的基础设施协议之一**，基于UDP/TCP（区域传送用TCP，查询常用UDP），端口53。负责域名到IP地址的解析，需掌握迭代查询与递归查询、域名服务器层次结构。 |
| **DHCP (动态主机配置协议)** | 基于UDP，为客户机动态分配IP地址及其他网络配置（子网掩码、网关、DNS）。过程：发现、提供、请求、确认。 |
| **TELNET** | 远程终端协议，基于TCP，端口23。数据明文传输，已被SSH取代。 |
| **SSH (安全外壳协议)** | 用于安全远程登录和管理，基于TCP，端口22。 |
| **SNMP (简单网络管理协议)** | 用于网络管理，基于UDP，端口161/162。 |

**考查重点**：HTTP协议细节、DNS解析过程、电子邮件系统组成与协议协作、TCP/UDP端口号。
